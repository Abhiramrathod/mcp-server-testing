Getting Started

Install and start testing in under 5 minutes

Installation

<dependency>
    <groupId>com.testMcp.io</groupId>
    <artifactId>mcp-test-api</artifactId>
    <version>1.0.0</version>
    <scope>test</scope>
</dependency>
testImplementation 'com.testMcp.io:mcp-test-api:1.0.0'
Note: Only add mcp-test-api as dependency. All other modules are included transitively.

Quick Start Example

import mcp.toolkit.testing.framework.api.*;
import mcp.toolkit.testing.framework.api.model.*;

public class MyFirstTest {
    @Test
    public void testMcpServer() {
        // Connect to MCP server
        McpClient client = McpClient.connectTo("http://localhost:8080")
                .config(McpClientConfig.builder()
                        .timeout(Duration.ofSeconds(30))
                        .build())
                .initializeOnBuild()
                .build();

        // Test tool invocation
        McpToolResult result = client.tools()
                .callTool("calculator", Map.of("operation", "add", "a", 5, "b", 3))
                .assertSuccess()
                .assertTextContains("8");

        // Verify performance
        client.exchanges().assertAverageLatencyBelow(McpMethod.TOOLS_CALL, 500);
        
        client.close();
    }
}

Core Capabilities

Everything you need to test MCP servers comprehensively

Tools Testing

Discover and invoke MCP tools with strongly-typed results and comprehensive validation.

// List available tools
List<McpTool> tools = client.tools().listTools();
tools.forEach(tool -> {
    System.out.println("Tool: " + tool.name());
    System.out.println("Description: " + tool.description());
});

// Call a tool with arguments
McpToolResult result = client.tools()
        .callTool("calculator", Map.of(
            "operation", "multiply",
            "a", 10,
            "b", 5
        ))
        .assertSuccess()
        .assertTextContains("50");

// Access results
String text = result.firstText();
boolean isError = result.isError();

Resources Testing

List and read MCP resources with content validation and MIME type handling.

// List available resources
List<McpResource> resources = client.resources().listResources();

// Read resource content
McpResourceContent content = client.resources()
        .readResource("file://config.json")
        .assertNotEmpty();

// Access different content types
String text = content.firstText();
JsonNode json = content.firstJson();
String mimeType = content.uri();

Prompts Testing

Retrieve and inspect MCP prompts with argument validation and message analysis.

// List available prompts
List<McpPrompt> prompts = client.prompts().listPrompts();

// Get prompt with arguments
McpPromptResult prompt = client.prompts()
        .getPrompt("translate", Map.of("language", "en"))
        .assertNotEmpty()
        .assertUserTextContains("translate");

// Inspect messages
String description = prompt.description();
List<Message> messages = prompt.messages();

Performance Monitoring

Track RPC exchanges and assert on latency with percentile calculations.

// Make multiple calls
for (int i = 0; i < 100; i++) {
    client.tools().callTool("fast-operation", Map.of());
}

// Assert on average latency
client.exchanges()
        .assertAverageLatencyBelow(McpMethod.TOOLS_CALL, 500);

// Calculate percentiles
long p50 = client.exchanges().latencyPercentile(McpMethod.TOOLS_CALL, 50);
long p95 = client.exchanges().latencyPercentile(McpMethod.TOOLS_CALL, 95);
long p99 = client.exchanges().latencyPercentile(McpMethod.TOOLS_CALL, 99);

// Assert on percentile
client.exchanges()
        .assertLatencyPercentileBelow(McpMethod.TOOLS_CALL, 99, 1000);

Architecture

Clean, modular design with clear separation of concerns

Public API
mcp-test-api McpClient, ToolsClient, Models
Internal Layer
mcp-test-client RPC Implementation
Transport Layer
mcp-test-transport SSE Transport
Core Components
mcp-test-interfaces Core Interfaces
mcp-test-core Utilities
Design Principle: Users only import mcp-test-api. All internal modules are abstracted away and included transitively.
